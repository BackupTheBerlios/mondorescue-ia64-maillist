Hi,
 
I have this situation:
 
Server: HP RX2660 (Itanium) 
OS    : Linux Red Hat 5.1
Tape Drive: HP Model: C7438A with OBDR function
 
I have downloaded from mondorescue:
- mindi-2.0.4-1.rhel5.src.rpm
- mindi-busybox-1.7.3-1.rhel5.src.rpm
- mondo-2.2.7-1.rhel5.src.rpm
- afio-2.4.7-1.ia64.rpm
- buffer-1.19-1.ia64.rpm
 
Compiled all src:
- rpmbuild -ba /usr/src/*/SPECS/mondo.spec
- rpmbuild -ba /usr/src/*/SPECS/mindi.spec
- rpmbuild -ba /usr/src/*/SPECS/mindi-busybox.spec
 
and installed all rpm:
- rpm -ivh /DVD/3PTY/afio-2.4.7-1.ia64.rpm
- rpm -ivh /DVD/3PTY/buffer-1.19-1.ia64.rpm
- rpm -ivh /usr/src/*/RPMS/*/mondo*.*.rpm /usr/src/*/RPMS/*/mindi*.*.rpm
 
Then I have run the backup operation (OBDR):
# mondoarchive -Oto -l ELILO -d /dev/st0
 
At the end of backup with No errors, I try to exec the recovery
operation so I put the tape drive in OBDR mode but the magnetc tape
(with backup) is automatically expelled.
 
the tape drive correctly works.  I am sure because the same tape drive
has been used for the mondorescue recovery on ProLiant server with
success.
 
I suspect that the problem is the boot file
/var/cache/mindi/mondorescue.iso, because the file is too small:
 
-rw-r--r-- 1 root root 34816 Dec  2 12:38 mondorescue.iso
 
Can someone help me ?
 
I attach the mondoarchive.log file.
 
Thanks
